Property Binding

The property-binding component class contains a property called angularLogo which includes an image link.
A few different methods of one-way property binding are demonstrated below:


HTML: <img src="{{angularLogo}}">

HTML: <img [src]="angularLogo">

HTML: <img bind-src="angularLogo">

As you might notice, the first example of property binding uses Interpolation, however, this only works with strings.
The next section will explain why.

Let's examine the button below:

HTML: <button>A Button</button>
Adding the disabled attribute disables the button
HTML: <button disabled>A Button</button>
Setting the disabled attribute to false does nothing
HTML: <button disabled="false">A Button</button>
Using interpolation to set the disabled attribute to false, also does nothing
HTML: <button disabled="{{false}}">A Button</button>
Using another method of property binding, setting the disabled attribute to false now works
HTML: <button [disabled]="false">A Button</button>

    Important Information: HTML Attirbutes vs DOM Properties

The input field below uses the following HTML code: <input type="text" value="Tyler">

After inspecting the input field above:
- Typing $0.getAttribute('value') in the browser console returns "Tyler".
- Typing $0.value in the browser console returns "Tyler".

The input field below uses the same HTML code as above: <input type="text" value="Tyler">

Replace "Tyler" with the text "Hello"
After inspecting the input field above:
- Typing $0.getAttribute('value') in the browser console returns "Tyler".
- Typing $0.value in the browser console returns "Hello".


Attributes initialize DOM properties and then they are done. Property values can change; attribute values can't. When the browser renders the above button, it creates a corresponding DOM node with a value property initialized to "Tyler". When the user enters "Hello" into the input box, the DOM element value property becomes "Hello". But the HTML value attribute remains unchanged as you discover if you ask the input element about that attribute: input.getAttribute('value') returns "Tyler". The HTML attribute value specifies the initial value; the DOM value property is the current value.
