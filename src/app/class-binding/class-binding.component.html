Class Binding

The following CSS will be used:

.text-success { color: green; }
.text-failure { color: red; }
.text-special { font-style: italic; }

This header includes class="text-success"

This header includes class="text-failure"

This header includes class="text-special"


In the class-binding component, a public property named successClass has a value of "text-success"
We can bind the class using this property by doing the following:
<h4 [class]="successClass">

This header includes [class]="successClass"


What if we have both a class attribute and class binding in the same header? :
<h4 [class]="successClass" class="text-special">

This header includes both [class]=successClass and class="text-special"

As you can see, class binding takes precedence over the class attribute


The next example shows you can conditionally apply a class. In the class-binding component class, a boolean property, failed, has been created and initially set to true

This header includes [class.text-failure]="failed"


The next example will use the ngClass directive. In the class-binding component class, we create an object messageClasses. Along with this, we have a public properties, failed set equal to true, and special set equal to true.
The messageClasses looks something like this:
public messageClasses = {
"text-success": !this.failed,
"text-failure": this.failed,
"text-special": this.special
}
We then bind the object:
<h4 [class]="messageClasses">

This header includes [ngClass]="messageClasses"

Since both failed and special are true, those styles will affect the header
